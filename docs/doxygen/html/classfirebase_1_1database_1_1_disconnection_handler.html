Detailed Description

Allows you to register server-side actions to occur when the client disconnects. Each method you call (with the exception of Cancel) will queue up an action on the data that will be performed by the server in the event the client disconnects. To reset this queue, call Cancel().

A DisconnectionHandler is associated with a specific location in the database, as they are obtained by calling DatabaseReference::OnDisconnect().
