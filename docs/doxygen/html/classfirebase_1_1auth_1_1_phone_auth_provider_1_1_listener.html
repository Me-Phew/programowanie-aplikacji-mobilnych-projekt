virtual void firebase::auth::PhoneAuthProvider::Listener::OnCodeAutoRetrievalTimeOut ( const std::string & verification_id)
virtual

The timeout specified in VerifyPhoneNumber has expired.

Called once auto_verify_time_out_ms has passed. If using auto SMS retrieval, you can choose to block the UI (do not allow manual input of the verification code) until timeout is hit.

Note
This callback is called immediately on iOS, since iOS does not have auto-validation.
Parameters
[in]verification_idIdentify the transaction that has timed out.

virtual void firebase::auth::PhoneAuthProvider::Listener::OnCodeSent ( const std::string & verification_id,
const ForceResendingToken & force_resending_token )
virtual

SMS message with verification code sent to phone number.

Called immediately after Auth server sends a verification SMS. Once receiving this, you can allow users to manually input the verification code (even if you're also performing auto-verification). For user manual input case, get the SMS verification code from the user and then call GetCredential with the user's code.

Parameters
[in]verification_idPass to GetCredential along with the user-input verification code to complete the phone number verification flow.
[in]force_resending_tokenIf the user requests that another SMS message be sent, use this when you recall VerifyPhoneNumber.

virtual void firebase::auth::PhoneAuthProvider::Listener::OnVerificationCompleted ( PhoneAuthCredential credential)
pure virtual

Phone number auto-verification succeeded.

Called when,

  • auto-sms-retrieval has succeeded–flow (2) in PhoneAuthProvider
  • instant validation has succeeded–flow (3) in PhoneAuthProvider
Note
This callback is never called on iOS, since iOS does not have auto-validation. It is always called immediately in the stub desktop implementation, however, since it fakes immediate success.
Parameters
[in]credentialThe completed credential from the phone number verification flow.
