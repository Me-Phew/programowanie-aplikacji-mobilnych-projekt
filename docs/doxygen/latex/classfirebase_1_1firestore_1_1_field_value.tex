\doxysubsection{Detailed Description}
A field value represents variant datatypes as stored by \doxylink{classfirebase_1_1firestore_1_1_firestore}{Firestore}. 

\doxylink{classfirebase_1_1firestore_1_1_field_value}{Field\+Value} can be used when reading a particular field with \doxylink{classfirebase_1_1firestore_1_1_document_snapshot_a726989caf1c55d7d355af3d5bfa3f518}{Document\+Snapshot\+::\+Get()} or fields with \doxylink{classfirebase_1_1firestore_1_1_document_snapshot_a27c719118b16ca67f96634015863baa9}{Document\+Snapshot\+::\+Get\+Data()}. When writing document fields with \doxylink{classfirebase_1_1firestore_1_1_document_reference_ad4a7cf6a1e9a51cafd99d3f9aeaeaeb7}{Document\+Reference\+::\+Set()} or \doxylink{classfirebase_1_1firestore_1_1_document_reference_a0d8d5bd4ffe02567e582d0c09d56387c}{Document\+Reference\+::\+Update()}, it can also represent sentinel values in addition to real data values.

For a non-\/sentinel instance, you can check whether it is of a particular type with is\+\_\+foo() and get the value with foo\+\_\+value(), where foo can be one of null, boolean, integer, double, timestamp, string, blob, reference, geo\+\_\+point, array or map. If the instance is not of type foo, the call to foo\+\_\+value() will fail (and cause a crash).
