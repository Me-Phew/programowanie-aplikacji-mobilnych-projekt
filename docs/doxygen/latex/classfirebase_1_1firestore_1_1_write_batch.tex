\doxysubsection{Detailed Description}
A write batch is used to perform multiple writes as a single atomic unit. 

A \doxylink{classfirebase_1_1firestore_1_1_write_batch}{Write\+Batch} object provides methods for adding writes to the write batch. None of the writes will be committed (or visible locally) until \doxylink{classfirebase_1_1firestore_1_1_write_batch_af9c8e438f0d9d52d441c90d7cbad699f}{Commit()} is called.

Unlike transactions, write batches are persisted offline and therefore are preferable when you don\textquotesingle{}t need to condition your writes on read data.

\doxylink{classfirebase_1_1firestore_1_1_firestore}{Firestore} classes are not meant to be subclassed except for use in test mocks. Subclassing is not supported in production code and new SDK releases may break code that does so. 
\end{DoxyNote}
