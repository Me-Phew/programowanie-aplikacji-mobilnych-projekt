\doxysubsection{Detailed Description}
\doxylink{classfirebase_1_1database_1_1_database_reference}{Database\+Reference} represents a particular location in your \doxylink{classfirebase_1_1database_1_1_database}{Database} and can be used for reading or writing data to that \doxylink{classfirebase_1_1database_1_1_database}{Database} location.

This class is the starting point for all \doxylink{classfirebase_1_1database_1_1_database}{Database} operations. After you\textquotesingle{}ve initialized it with a URL, you can use it to read data, write data, and to create new \doxylink{classfirebase_1_1database_1_1_database_reference}{Database\+Reference} instances.

Sets the priority of this field, which controls its sort order relative to its siblings. 

In Firebase, children are sorted in the following order\+:
\begin{DoxyEnumerate}
\item First, children with no priority.
\item Then, children with numerical priority, sorted numerically in ascending order.
\item Then, remaining children, sorted lexicographically in ascending order of their text priority.
\end{DoxyEnumerate}

Children with the same priority (including no priority) are sorted by key\+: A. First, children with keys that can be parsed as 32-\/bit integers, sorted in ascending numerical order of their keys. B. Then, remaining children, sorted in ascending lexicographical order of their keys.

This is an asynchronous operation which takes time to execute, and uses \doxylink{classfirebase_1_1_future}{firebase\+::\+Future} to return its result.


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em priority} & Sort priority for this child relative to its siblings. The \doxylink{classfirebase_1_1_variant}{Variant} types accepted are Null, Int64, Double, and String. Other types will return k\+Error\+Invalid\+Variant\+Type.\\

\end{DoxyParams}
