{\footnotesize\ttfamily \label{namespacefirebase_1_1database_a7534c9a8974d185b30280684addfcbe9} 
typedef \mbox{\hyperlink{namespacefirebase_1_1database_a9809aaddcfabafdeedee53895914704b}{Transaction\+Result}}(\texorpdfstring{$\ast$}{*} firebase\+::database\+::\+Do\+Transaction\+With\+Context) (\mbox{\hyperlink{classfirebase_1_1database_1_1_mutable_data}{Mutable\+Data}} \texorpdfstring{$\ast$}{*}data, void \texorpdfstring{$\ast$}{*}context)}

Your own transaction handler, which the Firebase Realtime \doxylink{classfirebase_1_1database_1_1_database}{Database} library may call multiple times to apply changes to the data, and should return success or failure depending on whether it succeeds. The context you specified to Run\+Transaction will be passed into this call.

This function will be called, {\itshape possibly multiple times}, with the current data at this location. The function is responsible for inspecting that data and modifying it as desired, then returning a Transaction\+Result specifying either that the \doxylink{classfirebase_1_1database_1_1_mutable_data}{Mutable\+Data} was modified to a desired new state, or that the transaction should be aborted. Whenever this function is called, the \doxylink{classfirebase_1_1database_1_1_mutable_data}{Mutable\+Data} passed in must be modified from scratch.

Since this function may be called repeatedly for the same transaction, be extremely careful of any side effects that may be triggered by this function. In addition, this function is called from within the Firebase Realtime \doxylink{classfirebase_1_1database_1_1_database}{Database} library\textquotesingle{}s run loop, so care is also required when accessing data that may be in use by other threads in your application.

Best practices for this function are to ONLY rely on the data passed in.


\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em data} & Mutable data, which the callback can edit. \\
\hline
\mbox{\texttt{ in}}  & {\em context} & Context pointer, passed verbatim to the callback.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
The callback should return k\+Transaction\+Result\+Success if the data was modified, or k\+Transaction\+Result\+Abort if it was unable to modify the data. If the callback returns k\+Transaction\+Result\+Abort, the Run\+Transaction() call will return the k\+Error\+Transaction\+Aborted\+By\+User error code.
\end{DoxyReturn}
\begin{DoxyNote}{Note}
If you want a callback to be triggered when the transaction is finished, you can use the Future$<$\+Data\+Snapshot$>$ value returned by the method running the transaction, and call \doxylink{classfirebase_1_1_future_adeaf5d44738aecc8aa49857bfa64625d}{Future\+::\+On\+Completion()} to register a callback to be called when the transaction either succeeds or fails.
\end{DoxyNote}
