The Firebase C++ SDK provides C++ interfaces for the following Firebase services on {\itshape i\+OS} and {\itshape Android}\+:


\begin{DoxyItemize}
\item Firebase Analytics
\item Firebase App Check
\item Firebase Authentication
\item Firebase Cloud Messaging
\item Firebase Dynamic Links (deprecated SDK)
\item Cloud Firestore
\item Firebase Functions
\item Google Mobile Ads (with User Messaging Platform)
\item Firebase Installations
\item Firebase Instance ID (deprecated SDK)
\item Firebase Realtime Database
\item Firebase Remote Config
\item Firebase Storage
\end{DoxyItemize}\hypertarget{md_build_2windows_2x64_2extracted_2firebase__cpp__sdk__windows_2readme_autotoc_md2}{}\doxysection{\texorpdfstring{Desktop Workflow Implementations}{Desktop Workflow Implementations}}\label{md_build_2windows_2x64_2extracted_2firebase__cpp__sdk__windows_2readme_autotoc_md2}
The Firebase C++ SDK includes desktop workflow support for the following subset of Firebase features, enabling their use on Windows, OS X, and Linux\+:


\begin{DoxyItemize}
\item Firebase Authentication
\item Firebase App Check
\item Cloud Firestore
\item Firebase Functions
\item Firebase Remote Config
\item Firebase Realtime Database
\item Firebase Storage
\end{DoxyItemize}

This is a Beta feature, and is intended for workflow use only during the development of your app, not for publicly shipping code.\hypertarget{md_build_2windows_2x64_2extracted_2firebase__cpp__sdk__windows_2readme_autotoc_md3}{}\doxysection{\texorpdfstring{Stub Implementations}{Stub Implementations}}\label{md_build_2windows_2x64_2extracted_2firebase__cpp__sdk__windows_2readme_autotoc_md3}

On i\+OS, some application events (such as opening URLs and receiving notifications) require your application delegate to implement specific methods. For example, receiving a notification may require your application delegate to implement {\ttfamily application\+:did\+Receive\+Remote\+Notification\+:}. Because each i\+OS application has its own app delegate, Firebase uses {\itshape method swizzling}, which allows the replacement of one method with another, to attach its own handlers in addition to any you may have implemented.

The Firebase Cloud Messaging library needs to attach handlers to the application delegate using method swizzling. If you are using these libraries, at load time, Firebase will identify your {\ttfamily App\+Delegate} class and swizzle the required methods onto it, chaining a call back to your existing method implementation.\hypertarget{md_build_2windows_2x64_2extracted_2firebase__cpp__sdk__windows_2readme_autotoc_md21}{}\doxysubsection{\texorpdfstring{Custom Android Build Systems}{Custom Android Build Systems}}\label{md_build_2windows_2x64_2extracted_2firebase__cpp__sdk__windows_2readme_autotoc_md21}

Many Firebase C++ libraries require \href{https://developers.google.com/android/guides/overview}{\texttt{ Google Play services}} on the user\textquotesingle{}s Android device.
