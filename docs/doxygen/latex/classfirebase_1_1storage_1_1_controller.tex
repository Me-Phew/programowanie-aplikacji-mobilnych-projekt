\doxysubsection{Detailed Description}
Controls an ongoing operation, allowing the caller to Pause, Resume or Cancel an ongoing download or upload. 

An instance of \doxylink{classfirebase_1_1storage_1_1_controller}{Controller} can be constructed and passed to \doxylink{classfirebase_1_1storage_1_1_storage_reference_a606eae171fe4dda094bdafde307e5022}{Storage\+Reference\+::\+Get\+Bytes()}, \doxylink{classfirebase_1_1storage_1_1_storage_reference_a77102fc20b083b13a3e9482d1b4b4678}{Storage\+Reference\+::\+Get\+File()}, \doxylink{classfirebase_1_1storage_1_1_storage_reference_a3da5eae65c4858dd68281f69bc1239a7}{Storage\+Reference\+::\+Put\+Bytes()}, or \doxylink{classfirebase_1_1storage_1_1_storage_reference_a1c1990b1936b13337ac55aa26f40db93}{Storage\+Reference\+::\+Put\+File()} to become associated with it. Each \doxylink{classfirebase_1_1storage_1_1_controller}{Controller} can only be associated with one operation at a time.

A \doxylink{classfirebase_1_1storage_1_1_controller}{Controller} is also passed as an argument to \doxylink{classfirebase_1_1storage_1_1_listener}{Listener}\textquotesingle{}s callbacks. The \doxylink{classfirebase_1_1storage_1_1_controller}{Controller} passed to a \doxylink{classfirebase_1_1storage_1_1_storage_reference}{Storage\+Reference} operation is not the same object passed to \doxylink{classfirebase_1_1storage_1_1_listener}{Listener} callbacks (though it refers to the same operation), so there are no restrictions on the lifetime of the \doxylink{classfirebase_1_1storage_1_1_controller}{Controller} the user creates (but the \doxylink{classfirebase_1_1storage_1_1_controller}{Controller} passed into a \doxylink{classfirebase_1_1storage_1_1_listener}{Listener} callbacks should only be used from within that callback).

This class is currently not thread safe and can only be called on the main thread.
