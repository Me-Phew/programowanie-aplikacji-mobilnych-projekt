\doxysubsection{Detailed Description}
Use phone number text messages to authenticate. 

Allows developers to use the phone number and SMS verification codes to authenticate a user on a mobile device.

This class is not supported on tv\+OS and Desktop platforms.

The verification flow results in a \doxylink{classfirebase_1_1auth_1_1_credential}{Credential} that can be used to,
\begin{DoxyItemize}
\item Sign in to an existing phone number account/sign up with a new phone number
\item Link a phone number to a current user. This provider will be added to the user.
\item Update a phone number on an existing user.
\item Re-\/authenticate an existing user. This may be needed when a sensitive operation requires the user to be recently logged in.
\end{DoxyItemize}

Possible verification flows\+: (1) \doxylink{classfirebase_1_1auth_1_1_user}{User} manually enters verification code.  $<$\+SWIG$>$  $<$/\+SWIG$>$

(2) SMS is automatically retrieved (Android only).

\item \doxylink{classfirebase_1_1_app}{App} calls \doxylink{classfirebase_1_1auth_1_1_phone_auth_provider_ad8b24001f507a420a3915d0f29865d84}{Verify\+Phone\+Number} with {\ttfamily timeout\+\_\+ms} \texorpdfstring{$>$}{>} 0.
\item \doxylink{classfirebase_1_1auth_1_1_auth}{Auth} server sends the verification code via SMS to the provided phone number.
\item SMS arrives and is automatically retrieved by the operating system. \doxylink{classfirebase_1_1auth_1_1_credential}{Credential} is automatically created and passed to the app via  $<$\+SWIG$>$  
\end{DoxyItemize}$<$/\+SWIG$>$
